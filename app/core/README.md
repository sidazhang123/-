# 核心基础目录

## 目录定位

`app/core/` 用于承载真正跨层复用的基础抽象。当前目录仍是预留结构，尚未沉淀出稳定的公共组件，因此只有 `__init__.py` 占位。

## 何时应把代码放进这里

1. 被 `api/`、`db/`、`services/` 同时依赖的公共协议或基础工具。
2. 不依赖具体业务领域，但会被多个子系统复用的基础能力。

## 何时不要放进这里

1. 与筛选任务、维护任务或概念任务强耦合的业务逻辑。
2. 仅服务某一个目录的辅助函数，这类代码应留在原目录内。

## 维护建议

1. 如果将来在这里新增模块，必须补全目录 README、模块级中文注释和用途边界。
2. 新增公共抽象后，要同步检查是否需要在 HANDOFF 或根 README 中增加“公共基础能力”说明。
