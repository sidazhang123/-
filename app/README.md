# 后端应用主目录

## 目录定位

`app/` 承载后端装配层与运行边界，是整个服务的应用入口。它负责把配置、状态库、数据访问层、业务服务层、API 路由层和静态页面组装成可运行的 FastAPI 服务。

## 当前结构

1. `main.py`：应用装配入口，初始化日志、状态库、任务管理器、维护管理器、概念管理器，并挂载 API 与静态资源。
2. `settings.py`：唯一配置读取与校验入口，负责把 `app_config.yaml` 转换为全局常量。
3. `app_config.yaml`：主配置文件，定义路径、并发、维护、概念抓取与日志保留策略。
4. `zsdtdx_config.yaml`：`zsdtdx` 运行配置，由启动阶段读取。
5. `api/`：HTTP 协议边界与响应模型拼装。
6. `db/`：状态库与源库访问逻辑。
7. `models/`：Pydantic 请求/响应模型。
8. `services/`：任务编排、维护执行、概念更新、策略注册等业务服务。
9. `static/`：监控页、结果页、维护页及其前端脚本。
10. `core/`：预留的公共基础目录，当前仅保留结构位。

## 关键边界

1. `app/` 不承载具体策略逻辑，策略实现统一放在 `strategies/groups/*`。
2. 配置项只能从 `app/settings.py` 导出，禁止在其他模块各自读取 YAML。
3. 新增概念任务、维护任务或筛选任务相关接口时，需要同步更新 `README.md`、`HANDOFF.md` 与对应子目录 README。

## 维护建议

1. 修改 `main.py` 时，优先关注应用生命周期、日志一致性和 `app.state` 注入对象是否同步调整。
2. 修改 `settings.py` 时，必须同时维护默认值、类型校验和配置文档。
3. 当子目录新增公开模块时，应同步补充目录 README、模块级中文注释与必要的函数级说明。
